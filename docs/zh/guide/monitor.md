# 实时监控

实时监控负责发现 STRM 的新增、修改、移动和删除，并按精确路径协调 Emby 更新；它不替代 MoviePilot、115 网盘 STRM 助手或神医助手的资源管理职责。

## 关键配置
![alt text](/public/images/ssjk.png)
- **监控路径**：需要监控的路径列表
- **排除路径**：必须位于监控路径内。命中后跳过入库前文件预处理，仅协调 Emby 入库与删除；确认入库后仍执行 Webhook 后处理
- **定时扫描回溯天数**：首次建立路径库存时，仅核对最近 N 天发生变化的 STRM，范围 1 至 7 天
- **全目录查漏间隔**：默认每 24 小时校准一次 STRM 路径库存，设为 0 可关闭；它不是每隔几分钟重复扫描全目录
- **监控扩展名**：仅监控配置了扩展名的文件类型
- **图片语言偏好**：元数据与图片预处理时将优先下载配置的语言图片

## 流程说明

普通路径的新文件会先执行 Toolkit 入库前预处理，再按精确路径通知 Emby。排除路径会跳过预处理，直接通知并确认 Emby 入库；确认条目后再复用 Webhook 持久队列完成中文化、图片和封面等后处理。新增任务不会递归刷新整个媒体库根目录。

Emby 在首次 8、20、45 秒核对后仍未收录的路径会进入 PostgreSQL 持久队列，约在第 10、30、60 分钟进行三轮后台重试。达到上限后自动停止，可在“STRM 入库诊断”中复制路径、手动重试或忽略，不会无限刷新。

删除和改名按完整文件路径处理：删除一集、删除整部电影/剧集目录、洗版导致文件名变化，都会分别向 Emby 发送旧路径删除和新路径新增。即使同一目录里存在名称接近的多个版本，也不会按标题合并或猜测。

## 常见建议

- 监控目录保持与 Emby 媒体库路径一致。
- MP 生成的 STRM 根目录同时填入“监控路径”和“排除路径”，让 MP 负责资源与 STRM，Toolkit 只负责发现变化并协调 Emby 入库。
- 不要监控临时下载/缓存目录，只监控已完成的媒体或 STRM 目录。
- 实时监控负责让 Emby 发现新文件；Emby Webhook 仍负责入库后的中文化、图片、封面和清理链。
- 任务中心的“同步媒体数据”仍是手动全量校准工具，不属于实时监控重试链。
