# 实时监控

实时监控新增媒体项。

## 关键配置
![alt text](/public/images/ssjk.png)
- **监控路径**：需要监控的路径列表
- **排除路径**：必须位于监控路径内。命中后跳过 Toolkit 刮削，仅协调 Emby 入库与删除
- **定时扫描回溯天数**：扫描监控目录任务将只扫描此配置天数内有变化的文件
- **STRM 自动查漏间隔**：周期核对排除路径中最近的 STRM 是否已被 Emby 收录，默认 15 分钟
- **监控扩展名**：仅监控配置了扩展名的文件类型
- **图片语言偏好**：刮削时将优先下载配置的语言图片

## 流程说明

普通路径的新文件会先生成 Toolkit 覆盖缓存，再按精确路径通知 Emby。排除路径会跳过这些刮削步骤，直接通知并确认 Emby 入库。新增任务不会递归刷新整个媒体库根目录。

## 常见建议

- 监控目录保持与 Emby 媒体库路径一致。
- MP 生成的 STRM 根目录同时填入“监控路径”和“排除路径”，让 MP 负责资源与 STRM，Toolkit 只负责发现变化并协调 Emby 入库。
- 不要监控临时下载/缓存目录，只监控已完成的媒体或 STRM 目录。
- 实时监控负责让 Emby 发现新文件；Emby Webhook 仍负责入库后的中文化、图片、封面和清理链。
